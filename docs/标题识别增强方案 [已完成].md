# 标题识别增强方案

## 问题描述

原有 `_extract_docx_segments` 函数在识别 Word 文档标题时存在以下问题：

1. **仅支持标准样式名**：只能识别 `Heading 1-9` 和 `标题 1-9` 两种标准样式
2. **无法识别自定义样式**：用户定义的"条款标题"、"章节名"等样式无法识别
3. **缺少正文编号兜底**：文档大量使用"第 X 章"、"第 X 条"、"（一）"、"一、"等编号，这些既不走样式名也不走大纲级别
4. **优先级不明确**：未明确多种识别方式的优先级顺序

## 解决方案

### 方案设计

采用**三级优先级识别策略**：

| 优先级 | 识别方式 | 支持范围 |
|-------|---------|---------|
| 1 | 标准样式（Heading n / 标题 n）> 自定义样式名 | 样式名匹配 |
| 2 | XML 样式 ID 兜底 | 某些未暴露到 `style.name` 的自定义样式 |
| 3 | 正文内容正则匹配 | 第 X 章、第 X 条、（一）、一、等编号模式 |

> 注：经技术调研，`python-docx` 库的 `paragraph_format` 不提供 `outline_level` 属性访问。Word 中的"大纲级别"实际通过样式定义体现，因此方案聚焦于样式名识别扩展 + 正文内容正则兜底。

### 实现细节

#### 1. 三级优先级识别策略

```python
def _get_heading_level(self, paragraph: Paragraph) -> int | None:
    # 优先级 1：从样式名识别（包含标准样式和自定义样式名）
    style_name = paragraph.style.name if paragraph.style else ""
    level = self._parse_heading_level_by_name(style_name)
    if level is not None:
        return level

    # 优先级 2：从底层 XML 读取样式 ID
    xml_element = paragraph._element
    # ... XML 读取逻辑

    # 优先级 3：从正文内容正则匹配识别
    text = paragraph.text.strip()
    if text:
        level = self._parse_heading_level_by_content(text)
        if level is not None:
            return level

    return None
```

#### 2. 自定义样式名匹配规则

在 `_parse_heading_level_by_name` 方法中新增对金融保险文档常用样式名的支持：

```python
custom_heading_patterns = [
    r"^条款标题$",
    r"^章节名$",
    r"^章$",
    r"^节$",
    r"^条$",
    r"^款$",
    r"^项$",
]
```

这些样式名统一视为**级别 1**标题，支持后续层级叠加。

#### 3. 正文内容正则匹配规则

新增 `_parse_heading_level_by_content` 方法，支持以下正文编号模式：

| 模式 | 示例 | 级别 |
|-----|------|-----|
| 第 X 章 | "第一章 总则"、"第 1 章" | 1 |
| 第 X 节 | "第一节 适用范围"、"第 1 节" | 2 |
| 第 X 条（中文数字） | "第一条 保险责任" | 2 |
| 第 X 条（阿拉伯数字） | "第 1 条 保险责任" | 3 |
| （一）、(一) | "（一）适用范围" | 2 |
| 一、二、三、 | "一、总则"、"十、附则" | 1 |
| 小数点分隔 | "1.1"、"1.1.1" | 根据段数推断 |

```python
def _parse_heading_level_by_content(self, text: str) -> int | None:
    # 模式 1：第 X 章、第 X 节、第 X 条
    chapter_match = re.match(r"^第\s*([一二三四五六七八九十\d]+)\s*(章 | 节 | 条)\b", text)

    # 模式 2：（一）、(一) —— 中文括号 + 中文数字
    paren_match = re.match(r"^[（(]([一二三四五六七八九十]+)[)）]", text)

    # 模式 3：一、二、三、 —— 中文数字 + 顿号
    chinese_num_match = re.match(r"^([一二三四五六七八九十]+)[、．.]", text)

    # 模式 4：1.1、1.1.1 —— 小数点分隔的数字
    dotted_match = re.match(r"^(\d+)(?:\.(\d+))?(?:\.(\d+))?... ", text)
```

#### 4. 方法职责重构

| 方法名 | 职责 |
|-------|------|
| `_get_heading_level(paragraph)` | 统一入口，三级优先级识别 |
| `_parse_heading_level_by_name(style_name)` | 基于样式名解析级别（标准 + 自定义） |
| `_parse_heading_level_by_content(text)` | 基于正文内容正则匹配解析级别 |
| `_parse_heading_level(style_name)` | 向后兼容的废弃方法，委托给 `_parse_heading_level_by_name` |

## 使用示例

### 场景 1：自定义样式名

用户在 Word 中使用自定义样式后，切块结果将正确保留层级上下文：

```
文档结构：
  条款标题：保险责任
    正文：本公司承担...
  章节名：责任免除
    正文：下列情况...

切块结果：
  - "条款标题"
  - "条款标题\n本公司承担..."
  - "条款标题 / 章节名"
  - "条款标题 / 章节名\n下列情况..."
```

### 场景 2：正文编号模式

用户未使用样式，仅通过正文编号组织文档：

```
文档结构：
  第一章 总则
  第一条 保险责任
    正文：本公司承担...
  （一）责任免除
    正文：下列情况...

切块结果：
  - "第一章 总则"
  - "第一章 总则 / 第一条 保险责任"
  - "第一章 总则 / 第一条 保险责任\n本公司承担..."
  - "第一章 总则 / 第一条 保险责任 / （一）责任免除"
  - "第一章 总则 / 第一条 保险责任 / （一）责任免除\n下列情况..."
```
