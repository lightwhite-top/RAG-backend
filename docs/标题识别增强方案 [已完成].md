# 标题识别增强方案

## 问题描述

原有 `_extract_docx_segments` 函数在识别 Word 文档标题时存在以下问题：

1. **仅支持标准样式名**：只能识别 `Heading 1-9` 和 `标题 1-9` 两种标准样式
2. **无法识别自定义样式**：用户定义的"条款标题"、"章节名"、"章"、"节"、"条"等样式无法识别
3. **标题上下文丢失**：正文 chunk 中缺少"这段文字属于哪个章节"的信息
4. **检索层级关系缺失**：检索时无法利用文档层级关系进行定位和权重提升

## 解决方案

### 方案设计

采用**样式名匹配扩展**方案，通过以下方式增强标题识别能力：

| 识别方式 | 优先级 | 支持范围 |
|---------|--------|---------|
| 样式名匹配（扩展） | 高 | 标准标题 + 自定义样式名 |
| XML 样式 ID 读取 | 兜底 | 某些未暴露到 `style.name` 的自定义样式 |

> 注：经技术调研，`python-docx` 库的 `paragraph_format` 不提供 `outline_level` 属性访问。Word 中的"大纲级别"实际通过样式定义体现，因此方案聚焦于样式名识别扩展。

### 实现细节

#### 1. 新增自定义样式名匹配规则

在 `_parse_heading_level_by_name` 方法中新增对金融保险文档常用样式名的支持：

```python
custom_heading_patterns = [
    r"^条款标题$",
    r"^章节名$",
    r"^章$",
    r"^节$",
    r"^条$",
    r"^款$",
    r"^项$",
]
```

这些样式名统一视为**级别 1**标题，支持后续层级叠加。

#### 2. 新增 XML 兜底读取逻辑

在 `_get_heading_level` 方法中，当 `style.name` 无法识别时，通过底层 XML 读取样式 ID：

```python
xml_element = paragraph._element
p_pr = xml_element.pPr
p_style = p_pr.pStyle
style_val = p_style.val
```

#### 3. 方法职责重构

| 方法名 | 职责 |
|-------|------|
| `_get_heading_level(paragraph)` | 统一入口，协调样式名匹配和 XML 兜底读取 |
| `_parse_heading_level_by_name(style_name)` | 基于样式名解析级别（标准 + 自定义） |
| `_parse_heading_level(style_name)` | 向后兼容的废弃方法，委托给 `_parse_heading_level_by_name` |

## 使用示例

用户在 Word 中使用自定义样式后，切块结果将正确保留层级上下文：

```
文档结构：
  条款标题：保险责任
    正文：本公司承担...
  章节名：责任免除
    正文：下列情况...

切块结果：
  - "条款标题"
  - "条款标题\n本公司承担..."
  - "条款标题 / 章节名"
  - "条款标题 / 章节名\n下列情况..."
```
